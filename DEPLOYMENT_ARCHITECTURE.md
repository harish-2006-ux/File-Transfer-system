## 🎯 Deployment Strategy

- **What:** Network Dashboard UI
- **Framework:** React + Vite
- **Location:** `frontend/` folder
- **URL:** `https://vaultx-frontend.vercel.app`

### **Backend (Flask)** → Render ✅
- **What:** File transfer API + Main app
- **Framework:** Flask + Python
- **Location:** Root directory

## 🚀 Deployment Steps

### Step 1: Deploy Backend to Render

**Platform:** Render (https://render.com)

**Why Render?**
- ✅ Perfect for Flask/Python apps
- ✅ Persistent disk for file storage
- ✅ Free tier available
- ✅ Auto-deploys from GitHub

**Configuration:**
```yaml
# render.yaml (already configured)
services:
  - type: web
    name: vaultx-backend
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn app:app
```

**Steps:**
1. Go to: https://dashboard.render.com
2. Click: "New" → "Blueprint"
3. Connect: Your GitHub repo
4. Render auto-detects: `render.yaml`
5. Add environment variables:
   ```
   SENDER_EMAIL=hhareeshvm@gmail.com
   SENDER_PASSWORD=mzkrjfaxdxnkrley
   DATABASE_TYPE=sqlite
   DEBUG=False
   ```
6. Click: "Apply"
7. Wait: 5-10 minutes

**Result:** `https://vaultx-backend.onrender.com`

---

### Step 2: Deploy Frontend to Vercel

**Platform:** Vercel (https://vercel.com)

**Why Vercel?**
- ✅ Perfect for React/Vite apps
- ✅ Global CDN
- ✅ Instant deployments
- ✅ Free tier available

**Configuration:**
```json
// vercel.json (already configured)
{
  "buildCommand": "cd frontend && npm install && npm run build",
  "outputDirectory": "frontend/dist",
  "framework": "vite"
}
```

**Steps:**
1. Go to: https://vercel.com/new
2. Import: Your GitHub repo
3. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Click: "Deploy"
5. Wait: 2-3 minutes

**Result:** `https://vaultx-frontend.vercel.app`

## ⚠️ Common Issues & Fixes

### Issue 1: Vercel tries to build Python backend

**Error:** `npm ERR! ERESOLVE could not resolve`

**Cause:** Vercel detecting wrong project type

**Fix:**
- ✅ Ensure `vercel.json` points to `frontend/` folder
- ✅ Set Root Directory to `frontend` in Vercel dashboard
- ✅ Remove any `node_modules/` from root directory

### Issue 2: Render dependency conflict

**Error:** `httpx version conflict`

**Cause:** `postgrest-py` conflicts with `supabase`

**Fix:**
- ✅ Remove `postgrest-py` from `requirements.txt`
- ✅ Keep only `supabase==2.3.4`
- ✅ Already fixed in latest commit!

### Issue 3: Flask app not accessible

**Error:** `Connection refused`

**Cause:** App listening on `127.0.0.1` instead of `0.0.0.0`

**Fix:**
- ✅ Update `app.py` to use `host="0.0.0.0"`
- ✅ Already fixed in latest commit!
