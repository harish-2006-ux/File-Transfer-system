# VaultX Deployment Guide

## 🚀 Deploy to Render (Backend) + Vercel (Frontend)

### Prerequisites
- GitHub account with your repository
- Render account (https://render.com)
- Vercel account (https://vercel.com)

---

## Part 1: Deploy Flask Backend to Render

### Option A: Using render.yaml (Recommended)

1. **Go to Render Dashboard**
   - Visit: https://dashboard.render.com
   - Click "New" → "Blueprint"

2. **Connect Repository**
   - Connect your GitHub account
   - Select repository: `File-Transfer-system`
   - Render will auto-detect `render.yaml`

3. **Configure Environment Variables**
   Add these in Render Dashboard:
   ```
   SECRET_KEY=<auto-generated>
   ENCRYPTION_KEY=<auto-generated>
   DATABASE_TYPE=sqlite
   DEBUG=False
   
   # Email (Optional)
   SENDER_EMAIL=your-email@gmail.com
   SENDER_PASSWORD=your-app-password
   
   # Supabase (Optional)
   SUPABASE_URL=your-supabase-url
   SUPABASE_KEY=your-supabase-key
   ```

4. **Deploy**
   - Click "Apply"
   - Wait 5-10 minutes for deployment
   - Your backend URL: `https://vaultx-backend.onrender.com`

### Option B: Manual Setup

1. **Create New Web Service**
   - Go to Render Dashboard
   - Click "New" → "Web Service"
   - Connect your GitHub repository

2. **Configure Service**
   ```
   Name: vaultx-backend
   Environment: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn app:app
   ```

3. **Add Environment Variables** (same as above)

4. **Add Persistent Disk** (for file storage)
   - Name: `vaultx-data`
   - Mount Path: `/opt/render/project/src/server_files`
   - Size: 1 GB

5. **Deploy** - Click "Create Web Service"

## Part 2: Deploy React Frontend to Vercel

### Method 1: Using Vercel CLI (Fastest)

1. **Install Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **Login to Vercel**
   ```bash
   vercel login
   ```

3. **Deploy Frontend**
   ```bash
   cd frontend
   vercel --prod
   ```

4. **Follow Prompts**
   - Set up and deploy: Yes
   - Which scope: Your account
   - Link to existing project: No
   - Project name: vaultx-frontend
   - Directory: `./` (current directory)
   - Override settings: No

### Method 2: Using Vercel Dashboard

1. **Go to Vercel Dashboard**
   - Visit: https://vercel.com/dashboard
   - Click "Add New" → "Project"

2. **Import Repository**
   - Connect GitHub
   - Select: `File-Transfer-system`
   - Click "Import"

3. **Configure Project**
   ```
   Framework Preset: Vite
   Root Directory: frontend
   Build Command: npm run build
   Output Directory: dist
   Install Command: npm install
   ```

4. **Environment Variables** (if needed)
   ```
   VITE_API_URL=https://vaultx-backend.onrender.com
   ```

5. **Deploy**
   - Click "Deploy"
   - Wait 2-3 minutes
   - Your frontend URL: `https://vaultx-frontend.vercel.app`

## Part 3: Connect Frontend to Backend

### Update Frontend API URL

If your frontend needs to connect to the backend, update the API URL:

**In `frontend/src/NetworkDashboard.jsx`** (or wherever API calls are made):

```javascript
// Replace localhost with your Render backend URL
const API_URL = 'https://vaultx-backend.onrender.com';
```

### Enable CORS on Backend

The Flask app already has CORS enabled for SocketIO. If you need additional CORS:

**In `app.py`**, add:
```python
from flask_cors import CORS
CORS(app, origins=['https://vaultx-frontend.vercel.app'])
```

Then add to `requirements.txt`:
```
flask-cors==4.0.0
```

## Part 4: Post-Deployment Configuration

### 1. Update Environment Variables on Render

Go to your Render service → Environment → Add:
- `SENDER_EMAIL` - Your Gmail
- `SENDER_PASSWORD` - Gmail App Password
- `SUPABASE_URL` - (if using Supabase)
- `SUPABASE_KEY` - (if using Supabase)

### 2. Test Your Deployment

**Backend Test:**
```bash
curl https://vaultx-backend.onrender.com/
```

**Frontend Test:**
- Visit: `https://vaultx-frontend.vercel.app`
- Should see the Network Dashboard

### 3. Custom Domain (Optional)

**Render:**
- Go to Settings → Custom Domain
- Add your domain

**Vercel:**
- Go to Settings → Domains
- Add your domain

## 🔧 Troubleshooting

### Backend Issues

**Problem:** App crashes on startup
- Check Render logs: Dashboard → Logs
- Verify all environment variables are set
- Check `requirements.txt` is complete

**Problem:** File uploads fail
- Ensure persistent disk is mounted
- Check disk space in Render dashboard

**Problem:** Database errors
- SQLite works on Render with persistent disk
- Or switch to Supabase for cloud database

### Frontend Issues

**Problem:** Build fails
- Check Node version (should be 18+)
- Verify `package.json` has all dependencies
- Check build logs in Vercel

**Problem:** API calls fail
- Update API URL to Render backend
- Check CORS settings
- Verify backend is running

**Problem:** 404 errors
- Check `vercel.json` routing configuration
- Ensure `dist` folder is being deployed

## 📝 Notes

1. **Free Tier Limitations:**
   - Render: Service sleeps after 15 min inactivity
   - First request after sleep takes 30-60 seconds
   - Consider paid plan for production

2. **Database:**
   - SQLite works with Render's persistent disk
   - For better performance, use Supabase (free tier available)

3. **File Storage:**
   - Render persistent disk: 1 GB free
   - For more storage, upgrade plan or use cloud storage (S3, Cloudinary)

4. **Environment Variables:**
   - Never commit `.env` file
   - Set all secrets in Render/Vercel dashboards
   - Use `.env.example` as reference
