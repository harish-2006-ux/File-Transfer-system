## 🚀 Deploy Now!

### Deploy Backend to Render:

1. **Go to:** https://dashboard.render.com
2. **Click:** "New" → "Blueprint"
3. **Select:** Your GitHub repository
4. **Render detects:** `render.yaml` automatically
5. **Add environment variables:**
   ```
   SENDER_EMAIL=hhareeshvm@gmail.com
   SENDER_PASSWORD=mzkrjfaxdxnkrley
   DATABASE_TYPE=sqlite
   DEBUG=False
   ```
6. **Click:** "Apply"
7. **Wait:** 5-10 minutes for deployment

**Your backend URL:** `https://vaultx-backend.onrender.com`

---

### Deploy Frontend to Vercel:

1. **Go to:** https://vercel.com/new
2. **Import:** Your GitHub repository
3. **Configure:**
   - Framework: Vite
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. **Click:** "Deploy"
5. **Wait:** 2-3 minutes

**Your frontend URL:** `https://vaultx-frontend.vercel.app`

## 🔧 Troubleshooting

### Issue: App won't start on Render
**Solution:** Check Render logs for errors
- Verify all environment variables are set
- Check `requirements.txt` is complete
- Ensure `Procfile` is correct

### Issue: Port binding error
**Solution:** Render automatically sets PORT environment variable
- Your app now reads from `os.getenv('PORT', 8000)`
- No manual configuration needed

### Issue: Database errors
**Solution:** 
- For SQLite: Ensure persistent disk is mounted
- For Supabase: Verify all 3 keys are set correctly
